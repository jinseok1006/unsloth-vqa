이 프로젝트는 VQA(Visual Question Answering) 멀티모달 모델을 파인튜닝하기 위한 실험용 저장소입니다.
주요 목표는 이미지와 객관식 질문을 함께 입력받아 정답을 a, b, c, d 중 한 글자로 예측하는 모델을 학습하는 것입니다.
현재는 Colab 환경에서 빠르게 실험을 반복할 수 있는 baseline 파이프라인을 정리하는 단계입니다.
기본 실행 환경은 Google Colab Pro+를 가정하고 있습니다.
파이썬 버전은 3.11 기준입니다.
주 사용 GPU는 A100 High-RAM 또는 상황에 따라 T4입니다.
프로젝트의 메인 작업 파일은 `jinseok.py` 입니다.
이 파일은 jupytext로 notebook과 동기화되는 `.py` 스크립트입니다.
따라서 실제 수정은 `.ipynb`가 아니라 `jinseok.py`에서 진행하는 것이 원칙입니다.
참고용 구현 파일로는 `train_unsloth_qwen35_9b_colab.py`가 있습니다.
모델 학습은 Unsloth 기반 Vision SFT 흐름을 사용합니다.
현재 기본 모델은 `unsloth/Qwen3-VL-2B-Instruct-unsloth-bnb-4bit` 입니다.
LoRA 기반 파인튜닝 설정을 사용하고 있으며, vision/language layer를 함께 미세조정합니다.
데이터는 `train.csv`, `test.csv`, `sample_submission.csv`, `train/`, `test/` 구조를 가정합니다.
원본 zip 파일은 Google Drive에 두고, Colab 세션에서 `/content` 아래로 압축 해제해서 사용합니다.
실제 학습은 Drive가 아니라 `/content`의 로컬 스토리지를 기준으로 진행합니다.
출력 산출물은 체크포인트, validation 결과, submission 파일 등을 포함합니다.
결과물은 세션 종료 전에 다시 Drive로 백업하는 흐름입니다.
질문 형식은 객관식이며, 선택지 a, b, c, d가 함께 제공됩니다.
프롬프트는 모델이 반드시 소문자 한 글자만 출력하도록 강하게 제한합니다.
현재 학습 데이터는 smoke test를 위해 subsample로 줄여서 사용하는 설정이 기본값입니다.
train/valid split도 코드 내부에서 단순하게 나누고 있습니다.
이미지 전처리는 현재 중요한 관심사 중 하나입니다.
최근에는 모든 원본 이미지를 무조건 처리하는 대신, 실제 학습/검증에 사용하는 subset만 전처리 캐시로 저장하도록 수정했습니다.
전처리 방식은 기본적으로 `keep_ratio_pad` 입니다.
즉 원본 비율은 유지하고, 남는 영역은 패딩해서 정사각형 입력으로 맞춥니다.
현재 기본 해상도는 `384x384` 입니다.
최근에는 중앙에 객체가 모여 있는 데이터 특성을 반영해, 전처리 단계에서 이미지 중심부를 아주 약하게 crop한 뒤 정사각형으로 맞추는 실험도 포함하고 있습니다.
전처리 결과는 `/content/preprocessed/{mode}_{size}` 아래에 저장되도록 정리했습니다.
또한 원본 이미지와 전처리 이미지를 랜덤 샘플로 비교해 볼 수 있는 preview 셀도 추가했습니다.
이 프로젝트에서 중요한 확인 포인트는 “설정한 해상도가 실제 학습 입력에 어떻게 반영되는가” 입니다.
특히 collator/processor 단계에서 추가 resize가 있는지 계속 점검하고 있습니다.
또 다른 이슈는 Colab에서 Google Drive zip 파일을 직접 읽어 압축 해제할 때 mount 불안정성으로 I/O 에러가 날 수 있다는 점입니다.
그래서 필요하면 zip을 먼저 `/content`로 복사한 뒤 압축 해제하는 방식으로 더 안정화할 계획이 있습니다.
현재 코드는 빠른 실험과 디버깅 편의성을 우선한 상태입니다.
즉 대규모 학습 최적화보다는, 데이터 경로/전처리/학습/검증 흐름이 명확히 보이도록 구성되어 있습니다.
향후 개선 포인트로는 전처리 파이프라인 안정화, 최종 입력 해상도 검증, validation 체계 개선, full inference 자동화가 있습니다.
요약하면 이 프로젝트는 Colab에서 Unsloth 기반 VQA 모델을 빠르게 실험하기 위한 notebook-style 학습 파이프라인입니다.
현재는 특히 이미지 전처리 캐시 전략과 실제 모델 입력 해상도 일치 여부를 중점적으로 다듬고 있습니다.
